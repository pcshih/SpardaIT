案例：公司多設備連線不穩，透過固定 IP 規劃改善網路管理與穩定性

統一規劃固定 IP，時間：2025-10-03

本次案例來自一家小型會計事務所，近期多位員工反映公司內部網路時常出現連線中斷、NAS 讀取延遲、印表機無法連線等問題。這些狀況雖然重開電腦後偶爾會恢復，但隔天又會再次發生。 由於業務大量依賴檔案存取與雲端報稅系統，頻繁的網路異常已影響到工作效率，因此我們希望能找出根本原因，並重新調整網路環境。

在抵達現場檢查後，我們先查看路由器與交換器的運作狀況，並針對主要電腦、NAS、印表機與 Wi-Fi AP 逐一確認連線設定。 檢測過程中發現，許多設備皆使用預設的 DHCP 自動取得 IP，而路由器發放的 IP 範圍又相當有限，加上辦公室近年增加了多台筆電與無線裝置，導致 IP 經常「重複」或「衝突」。

例如 NAS 原本應保持固定位置，但因 DHCP 自動分配，IP 在重新開機後改變，使得部分電腦找不到網路磁碟。 印表機也因 IP 變動導致無法連線，員工必須重新安裝或手動更新 IP 才能列印。這些症狀正是典型的 IP 配置混亂造成。

確認問題後，我們向客戶說明了導致不穩定的主因：設備數量增加後，DHCP 亂跳 IP 導致位址衝突，進而造成 NAS、印表機與共享資料夾時常失聯。 為了解決這類問題，我們建議採用「固定 IP 規劃」，將所有重要設備指定為固定 IP，並重新整理整個網路的位址配置。

經討論後，我們開始進行 IP 規劃，將網段劃分為三類： 第一類為核心設備（NAS、伺服器、交換器管理界面）； 第二類為辦公室固定電腦與重要印表機； 第三類則保留給 Wi-Fi 裝置與臨時訪客設備。 我們設定路由器的 DHCP 區間遠離上述固定 IP，使自動分配與手動指定不會互相重疊。

完成規劃後，我們逐台設定固定電腦的 IP，包括子網路遮罩、閘道及 DNS； 同時也調整 NAS 與印表機的網路參數，使它們永遠保持在固定位置。 為了確保未來的設備新增不會混淆，我們也替客戶建立 IP 分配紀錄表，清楚標示每個位址的用途。

設定完成後，我們進行連線測試，包括網路磁碟掛載、印表機列印測試、NAS 讀取速度測試以及局域網傳輸。 所有功能皆正常，且沒有出現位址衝突或不明斷線情形。 多台電腦同時讀取資料時，NAS 的反應也變得穩定許多。 客戶表示，過去常因找不到印表機或連不到 NAS 而浪費不少時間，現在這些問題完全消失，整體效率顯著提升。

本案例顯示，當企業內部設備逐漸增加時，單純依靠 DHCP 自動分配 IP 很容易形成混亂，甚至影響日常運作。 透過完整的固定 IP 規劃與配置，不僅能提升網路穩定性，也能減少維護上的困擾。 若發現共享資料、印表機或 NAS 常無預警斷線，通常與 IP 分配混亂有關，建議儘早由專業人員協助規劃，以確保辦公室網路環境維持最佳狀態。

➤ 維修結論：重新規劃並設定固定 IP，有效解決連線不穩與設備衝突問題，網路運作全面恢復順暢。
